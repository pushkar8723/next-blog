## The Problem

However this is where I was hit by another realization. My blog was not easily customizable. To use Gatsby, there
is a steep learning curve. Especially if you are not familiar with UI tech stack and GraphQL. I was not satisfied
the image tile that Gatsby Plugin provided. I wanted to add few more things like subtitle, date and reading time.
I was already exhausted from configuration I had to do to add projects, pages, configuring different Gatsby plugins
and making all of these work with each other, I didn‚Äôt had the motivation left in me to make more changes.
This was before the advent of AI, hence making any change took a long time.

Thus for a long time, I wanted to move to Next.js. File based routing is simpler than Gatsby's configuration
and using `generateStaticParams` is a lot easier than writing GraphQL queries. It comes with many optimizations
to create a static website. Only downside was that there is no rich plugin ecosystem like Gatsby. So I needed to
build a lot of features manually to match what `gatsby-starter-blog` provided.

## Enter v0

Today's landscape is very different from 2020. We have AI and tools like [Replit](https://replit.com/),
[Loveable](https://lovable.dev/) and [v0](https://v0.app/). I chose v0 to create Next.js based blog as it is part
of [Vercel](https://vercel.com/) ecosystem.

I entered a prompt with a comprehensive list of features that I wanted and let it do its magic. I did needed few
corrections here and there but overall it was a great experience. In half an hour I had a working prototype with
most of the features I wanted. I did want to improve few things but thought to get more opinions before doing that.
I shared this with my friends and they too had similar feedback to what I was thinking.
